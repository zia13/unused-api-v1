# API Gateway Cleanup - CSV-Based Manual Approval Workflow

## Overview

This automated solution identifies and decommissions unused AWS API Gateway APIs using a **CSV-based manual approval workflow**. The team reviews and approves each API deletion before any action is taken.

## Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│  EventBridge Scheduler (Weekly: Monday 08:00 UTC)               │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│  Lambda: Scanner                                                 │
│  • Scans all REST/HTTP/WebSocket APIs across all regions        │
│  • Pulls 90-day CloudWatch metrics (invocation count)           │
│  • Generates CSV: api-scan-YYYYMMDD.csv                         │
│  • Uploads to: s3://bucket/scans/                               │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│  Lambda: Classifier                                              │
│  • Downloads latest scan CSV from S3                            │
│  • Applies tier classification (ACTIVE/LOW_TRAFFIC/             │
│    DORMANT/ORPHANED)                                            │
│  • Generates CSV: api-classification-YYYYMMDD.csv               │
│  • Uploads to: s3://bucket/classifications/                     │
│  • Sends email notification to team with download link          │
│  • Sends individual notifications to API owners for             │
│    DORMANT/ORPHANED APIs with 30-day deletion warning           │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│  Manual Review by API Team                                       │
│  • Download classification CSV from S3                          │
│  • Review each API (especially DORMANT/ORPHANED)                │
│  • Fill in approval columns:                                    │
│    - ApprovalStatus: APPROVE_DELETE / KEEP / EXTEND_REVIEW      │
│    - ReviewerName: Your name                                    │
│    - ReviewerComments: Reason for decision                      │
│    - ReviewDate: Date of review                                 │
│  • Upload to: s3://bucket/approvals/api-approved-YYYYMMDD.csv  │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│  Jenkins Pipeline: Decommission                                  │
│  • Stage 1: Download & validate approved CSV                    │
│  • Stage 2: Backup OpenAPI specs to S3                          │
│  • Stage 3: Soft Delete (throttle to 0 req/s)                   │
│  • Stage 4: Manual approval (7-day wait)                        │
│  • Stage 5: Hard Delete (permanent deletion)                    │
│  • Stage 6: Generate report & send email                        │
└─────────────────────────────────────────────────────────────────┘
```

## CSV Schemas

### 1. Scanner Output CSV (`api-scan-YYYYMMDD-HHMMSS.csv`)

Generated by `lambda_scanner.py` and uploaded to `s3://bucket/scans/`

| Column | Description | Example |
|--------|-------------|---------|
| ScanDate | ISO8601 timestamp | `2026-04-28T08:00:00Z` |
| AccountId | AWS account ID | `123456789012` |
| AccountName | AWS account alias | `prod-account` |
| Region | AWS region | `us-east-1` |
| ApiId | API Gateway API ID | `abc123xyz` |
| ApiName | API name | `customer-api` |
| ApiType | `REST`, `HTTP`, or `WEBSOCKET` | `REST` |
| CreatedDate | API creation date | `2024-01-15T10:30:00Z` |
| HasStages | Boolean | `true` |
| StageNames | Comma-separated | `prod,dev` |
| HasCustomDomain | Boolean | `false` |
| HasUsagePlan | Boolean | `true` |
| InvocationCount90d | Total invocations (90 days) | `1523` |
| LastInvocationDate | Last traffic date | `2026-02-10T14:20:00Z` |
| AvgRequestsPerDay | Average daily requests | `16.9` |
| OwnerEmail | From `owner` tag | `john.doe@company.com` |
| TeamTag | From `team` tag | `platform-team` |
| Tags | JSON string of all tags | `{"owner":"..."}` |

### 2. Classifier Output CSV (`api-classification-YYYYMMDD-HHMMSS.csv`)

Generated by `lambda_classifier.py` and uploaded to `s3://bucket/classifications/`

Includes all Scanner CSV columns **plus**:

| Column | Description | Example |
|--------|-------------|---------|
| Tier | Classification tier | `DORMANT` |
| TierReason | Reason for classification | `Zero traffic for 45 days` |
| ClassifiedDate | Classification timestamp | `2026-04-28T08:15:00Z` |
| RecommendedAction | `DELETE` or `REVIEW` | `DELETE` |
| **ApprovalStatus** | **[TO BE FILLED BY TEAM]** | `APPROVE_DELETE` |
| **ReviewerName** | **[TO BE FILLED BY TEAM]** | `John Doe` |
| **ReviewerComments** | **[TO BE FILLED BY TEAM]** | `Confirmed with team` |
| **ReviewDate** | **[TO BE FILLED BY TEAM]** | `2026-04-29` |

### 3. Approved CSV (`api-approved-YYYYMMDD.csv`)

Manually updated by team and uploaded to `s3://bucket/approvals/`

Same as Classifier Output CSV with approval columns filled in.

**ApprovalStatus Values:**
- `APPROVE_DELETE` - API will be deleted by Jenkins pipeline
- `KEEP` - API will be retained (no action)
- `EXTEND_REVIEW` - Extend review by 30 days (re-scan next cycle)

### 4. Deletion Report CSV (`api-deletion-report-YYYYMMDD-HHMMSS.csv`)

Generated by Jenkins pipeline and uploaded to `s3://bucket/reports/`

Includes all Approved CSV columns **plus**:

| Column | Description | Example |
|--------|-------------|---------|
| BackupS3Key | S3 location of OpenAPI backup | `backups/.../oas30.json` |
| SoftDeleteDate | Date of throttle action | `2026-04-30T09:00:00Z` |
| HardDeleteDate | Date of deletion | `2026-05-07T09:00:00Z` |
| DeletionStatus | `SUCCESS` or `FAILED` | `SUCCESS` |
| DeletionError | Error message if failed | `` |
| JenkinsJobUrl | Jenkins job URL | `https://jenkins.../123` |

## Classification Tiers

| Tier | Criteria | Recommended Action |
|------|----------|-------------------|
| **ACTIVE** | Traffic in last 30 days | Keep (no action) |
| **LOW_TRAFFIC** | < 10 req/day average (configurable) | Review usage patterns |
| **DORMANT** | Zero traffic for 30+ days, has stages | **DELETE** |
| **ORPHANED** | No stages or never had traffic | **DELETE** |

## Setup Instructions

### 1. Deploy Lambda Functions

```bash
cd automation/api-gateway-cleanup/terraform
terraform init
terraform plan
terraform apply
```

This creates:
- Lambda: `api-scanner` (EventBridge scheduled)
- Lambda: `api-classifier` (triggered after scanner)
- S3 bucket: `api-gateway-cleanup-bucket`
- IAM roles with necessary permissions

### 2. Configure Email Notifications

Update Lambda environment variables:
```
SES_SENDER_EMAIL=noreply@company.com
NOTIFICATION_EMAILS=api-team@company.com,platform@company.com
```

Verify SES email addresses in AWS Console.

### 3. Configure Jenkins Job

1. Create new Pipeline job: `api-gateway-decommission`
2. Set Pipeline script from SCM:
   - Repository: This repo
   - Script Path: `automation/api-gateway-cleanup/Jenkinsfile-decommission`
3. Configure AWS credentials
4. Set up S3 event notification (optional):
   - Trigger Jenkins when file uploaded to `approvals/` prefix

## Usage Workflow

### Step 1: Automated Scan & Classification (Weekly)

Every Monday at 08:00 UTC:
1. Scanner Lambda runs → generates scan CSV
2. Classifier Lambda runs → generates classification CSV
3. Email sent to team with download link
4. **Individual emails sent to API owners** for DORMANT/ORPHANED APIs with:
   - List of their unused APIs
   - 30-day deletion warning
   - Actions they can take (confirm, request extension, or tag as protected)
   - 14-day response deadline

### Step 2: Manual Review (Team)

1. **Download** classification CSV from email link or S3:
   ```bash
   aws s3 cp s3://api-gateway-cleanup-bucket/classifications/api-classification-20260428-080000.csv .
   ```

2. **Open** in Excel/Google Sheets

3. **Review** each API, focusing on DORMANT and ORPHANED tiers

4. **Update** approval columns:
   - **ApprovalStatus**: `APPROVE_DELETE`, `KEEP`, or `EXTEND_REVIEW`
   - **ReviewerName**: Your name
   - **ReviewerComments**: Brief reason (e.g., "Deprecated service, safe to delete")
   - **ReviewDate**: Today's date (e.g., `2026-04-29`)

5. **Upload** to S3:
   ```bash
   aws s3 cp api-classification-20260428-080000.csv \
     s3://api-gateway-cleanup-bucket/approvals/api-approved-20260429.csv
   ```

### Step 3: Jenkins Decommission Pipeline

#### Option A: Manual Trigger

1. Go to Jenkins: https://jenkins.company.com/job/api-gateway-decommission
2. Click "Build with Parameters"
3. Enter parameters:
   - **APPROVED_CSV_S3_KEY**: `approvals/api-approved-20260429.csv`
   - **S3_BUCKET**: `api-gateway-cleanup-bucket`
   - **NOTIFICATION_EMAILS**: Your email(s)
   - **DRY_RUN**: `false` (or `true` for testing)
4. Click "Build"

#### Option B: Automatic Trigger (if configured)

Pipeline triggers automatically when CSV uploaded to `approvals/` prefix.

### Step 4: Pipeline Execution

The Jenkins pipeline will:

1. **Download & Validate** (1 min)
   - Downloads approved CSV
   - Validates format
   - Counts approved deletions

2. **Backup** (2-5 min)
   - Exports OpenAPI specs for all APIs
   - Uploads to S3 `backups/` prefix

3. **Soft Delete** (2-5 min)
   - Throttles all stages to 0 req/s
   - Detaches custom domains
   - APIs become inaccessible but not deleted

4. **Manual Approval** (7 days timeout)
   - Jenkins pauses for manual approval
   - Approvers: `api-admins`, `platform-team`
   - Time to verify soft delete didn't break anything

5. **Hard Delete** (2-5 min)
   - Permanently deletes APIs
   - Updates status in CSV

6. **Report** (1 min)
   - Generates deletion report
   - Uploads to S3 `reports/` prefix
   - Sends completion email

### Step 5: Verify Results

Check deletion report:
```bash
aws s3 ls s3://api-gateway-cleanup-bucket/reports/
aws s3 cp s3://api-gateway-cleanup-bucket/reports/api-deletion-report-20260430-090000.csv .
```

## Configuration

### Lambda Environment Variables

#### Scanner Lambda (`lambda_scanner.py`)
```
S3_BUCKET=api-gateway-cleanup-bucket
S3_SCAN_PREFIX=scans/
LOOKBACK_DAYS=90
```

#### Classifier Lambda (`lambda_classifier.py`)
```
S3_BUCKET=api-gateway-cleanup-bucket
S3_SCAN_PREFIX=scans/
S3_CLASSIFICATION_PREFIX=classifications/
LOOKBACK_DAYS=90
LOW_TRAFFIC_THRESHOLD=10
DORMANT_DAYS=30
SES_SENDER_EMAIL=noreply@company.com
NOTIFICATION_EMAILS=api-team@company.com,platform@company.com
```

### Jenkins Pipeline Parameters

- **APPROVED_CSV_S3_KEY**: S3 key of approved CSV (required)
- **S3_BUCKET**: S3 bucket name (default: `api-gateway-cleanup-bucket`)
- **NOTIFICATION_EMAILS**: Email recipients (default: `api-team@company.com`)
- **DRY_RUN**: Test mode without deletions (default: `false`)

## Troubleshooting

### Issue: Scanner Lambda returns 0 invocation counts

**Cause**: CloudWatch detailed metrics not enabled on API Gateway stages

**Solution**: Enable detailed CloudWatch metrics:
```bash
aws apigateway update-stage \
  --rest-api-id abc123 \
  --stage-name prod \
  --patch-operations op=replace,path=/metricsEnabled,value=true
```

### Issue: Classifier email not received

**Cause**: SES email addresses not verified

**Solution**: Verify email addresses in SES:
```bash
aws ses verify-email-identity --email-address api-team@company.com
```

### Issue: Jenkins pipeline fails at backup stage

**Cause**: API has no deployments or stages

**Solution**: Pipeline continues with warning. Check logs for specific API.

### Issue: Soft delete doesn't stop traffic

**Cause**: Custom domain or usage plan still routing traffic

**Solution**: Pipeline should detach custom domains. Check CloudWatch metrics.

## Security Considerations

1. **S3 Bucket Encryption**: Enable server-side encryption
2. **Access Logging**: Enable S3 and CloudWatch logging
3. **Least Privilege**: Use minimal IAM permissions
4. **Backup Retention**: Set lifecycle policy (e.g., 90 days)
5. **MFA Delete**: Enable MFA delete on S3 bucket
6. **Audit Trail**: All actions logged to CloudTrail
